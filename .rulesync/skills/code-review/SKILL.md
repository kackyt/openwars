# コードレビュースキル

このスキルは `main` ブランチとの差分を解析し、`.rulesync/rules/project.md` の規約に照らした**敵対的レビュー**を実施して、指摘をMarkdownにまとめます。

## 実行フロー

### 1. 差分の取得

```bash
# 現在のブランチ名を確認
git rev-parse --abbrev-ref HEAD

# mainとの差分を取得（コンテキスト付き）
git diff main...HEAD
```

差分が大きい場合はファイル単位で分割して取得：

```bash
# 変更されたファイル一覧
git diff main...HEAD --name-only

# ファイルごとの差分
git diff main...HEAD -- <filepath>
```

### 2. 規約の確認

`.rulesync/rules/project.md` を `view_file` で読み込み、以下の規約軸を把握する。

### 3. 敵対的レビューの実施

差分をファイルごとに解析し、下記のチェック観点で**厳しく、妥協なく**指摘する。
些細な違反でも見逃さないこと。

| 重要度 | チェック観点 | 根拠（rules/project.md） |
|--------|-------------|--------------------------|
| 🔴 重大 | エンティティへの参照をIDではなくポインタ・参照で保持している | ID参照の原則 |
| 🔴 重大 | ドメイン層・インフラ層で `anyhow` を直接使っている（`thiserror` を使うべき） | エラーハンドリング規約 |
| 🔴 重大 | アプリ層がインフラ層の具体型に直接依存している（Trait経由でない） | DI原則 |
| 🔴 重大 | ロジック変更にテストが伴っていない | テスト戦略 |
| 🔴 重大 | `engine/systems/` 以外(cli等)でゲームルールロジックを実装している | engineクレート構成規約 |
| 🟡 警告 | 日本語コメントがない、または意図がわからないコメントのみ | コードスタイル規約 |
| 🟡 警告 | 不必要な `.clone()` がある | メモリライフサイクル規約 |
| 🟡 警告 | `Box<dyn Trait>` のライフサイクルが不明瞭 | メモリライフサイクル規約 |
| 🟡 警告 | SOLID原則違反（単一責任・開放閉鎖など） | SOLID/DRY原則 |
| 🟡 警告 | DRY違反（重複ロジック） | SOLID/DRY原則 |
| 🟡 警告 | `engine/components/` に描画用データが混入している | engineクレート構成規約 |
| 🟡 警告 | プリミティブ型（`i32`, `String` 等）を直接フィールドに使用している | Newtypeパターンの徹底 |
| 🔵 提案 | `cargo clippy` / `cargo fmt` で検出される軽微な問題 | コードスタイル規約 |

### 4. レビュー結果のMarkdown出力

以下のフォーマットで `review_<ブランチ名>.md` をリポジトリルートに生成する。

```markdown
# コードレビュー: <ブランチ名>
> レビュー日時: <日時>
> 対象: `git diff main...<ブランチ名>`

## サマリー

| 重要度 | 件数 |
|--------|------|
| 🔴 重大 | N件 |
| 🟡 警告 | N件 |
| 🔵 提案 | N件 |

**総評**: （全体的な所感を2〜3文で厳しく記述）

---

## 指摘事項

### 🔴 [重大] `path/to/file.rs` - 指摘タイトル

**問題**:
（何が問題なのかを具体的に説明）

**該当コード**:
\```rust
// 問題のあるコードスニペット
\```

**根拠**: `rules/project.md` — 「...該当規約の引用...」

**修正案**:
\```rust
// 修正後のコードスニペット
\```

---

### 🟡 [警告] ...

---

## 問題なし

変更内容のうち、規約に準拠している箇所があれば簡潔に記載する。
```

### 5. 完了報告

`notify_user` でレビュー完了を報告する。指摘件数のサマリーと出力ファイルパスを伝える。

## 注意事項

- **厳しさを保つこと**: レビューは「敵対的」に行う。「まあいいか」は禁止。
- **根拠を必ず示すこと**: すべての指摘に `rules/project.md` の該当箇所を明示する。
- **差分外のファイルには言及しない**: 変更されていないコードは対象外。
- **差分が存在しない場合**: ユーザーに「mainブランチとの差分がありません」と報告して終了。
