# OpenWars MCP AI Evaluator

OpenWarsのMCPサーバー (`openwars`) を使用して、開発・連携中のAIの動作確認と評価を対話的に行うワークフローです。
このスキルは、新しく実装されたAIロジックが、期待される戦術や戦略に従ってユニットを操作し、勝利に向かってゲームをプレイできるかを検証・解析することを目的とします。

## 対象となるケース
- 新しいAIロジックや評価関数を実装した際のE2E評価
- 特定のマップ (`g1_01`, `g1_02` など) におけるAIの行動シミュレートと品質検証
- 現在のボード状態と、AIがどの行動を選択するか (Valid Actions から何を選ぶか) のデバッグ

## 評価ワークフロー

AIのプレイを評価するにあたり、以下の手順を順番に実行してください。

### 1. マップの問い合わせと初期化
まず、評価を行う対象のマップについてユーザーに問い合わせ、確認を取ってからマップをロードします。

- 実行するアクション: ユーザーに「どのマップで評価を行いますか？ (例: g1_01, g1_02)」と質問する。
- 実行するMCPツール: ユーザーからマップ名の回答を得た後、`load_map` を実行する。
- パラメータ: `map_name` (例: `g1_01`, `g1_02`)
- 目的: 意図した環境（AIと対戦、またはAI同士で対戦させるボード環境）を初期化します。

### 2. 初期状態の確認と評価指標の取得
ゲームが開始されたときの現在の盤面状況を把握します。

- 実行するMCPツール: `get_board_state` と `evaluate_board`
- 目的: 各勢力(Faction)のユニット配置、地形、拠点状況を確認し、初期状態におけるボードの優劣評価を記録します。

### 3. AIターンのシミュレート
AIのターンを連続的に実行し、プレイ内容を観察します。

- 実行するMCPツール: `simulate_ai_turn` (複数回)
- 併用ツール: AIの実行前後の盤面変化を見るために `get_board_state` 、また個別のユニットに対して `get_valid_actions` を呼び出して行動の選択肢を確認することも有益です。
- 目的: AIが敵を正しく攻撃しているか、自軍の拠点を占領しているか、無駄な移動をしていないかをログや盤面から確認します。

### 4. アクションの手動実行（デバッグが必要な場合）
AIの挙動が不完全な場合や手動で干渉してケースを再現したい場合は、行動を直接注入します。

- 実行するMCPツール: `execute_action`
- 目的: 特定の条件下（劣勢、優勢、接敵時など）でAIがどう挙動するかを見るためのセットアップを行います。

### 5. 評価結果・レポートの作成
シミュレーションが完了したら、結果をユーザー向けにMarkdownのレポートとして出力します。
レポートには以下の要素を含めてください:

- **実行マップ**: テストしたマップ名
- **発生したイベント**: 攻撃や生産、占領などのダイジェスト
- **戦術の評価**:
  - 資源管理（不要なユニット生成がないか等）
  - 戦闘効率（有利な地形で戦っているか、無闇な特攻をしていないか）
  - 目的意識（敵の首都や中立拠点に向かっているか）
- **改善提案**: 現在のAIロジックにおける明らかな弱点や直すべき問題点

## 注意事項

- コマンドラインツール（`run_command`）から手動でコマンドを叩くのではなく、MCPサーバー `openwars` で公開されているツール群を必ず利用して盤面にアクセスしてください。
- 各MCPツールのレスポンスが「Tool '...' is not wired to the engine yet」となっている場合は、まだMCP連携機能が未実装です。ユーザーにその旨を報告し、エンジンとの接続実装のタスクへ移行するか確認してください。
