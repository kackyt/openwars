## Context

現在、CLIのUI表示（メニュー、プロンプト、ヘルプ、ログ）は英語で実装されています。一方で、エンジン層のデータ（ユニット種別や地形名）は既に日本語化されています。この乖離を解消し、CLI全体を日本語で統一します。

## Goals / Non-Goals

**Goals:**
- CLIの全UI表示を日本語化する。
- ソースコード内のハードコードされた英語文字列を日本語に置き換える。
- ユーザー操作（キー入力のヘルプなど）の日本語化。
- **武器名の表示**: ユニット詳細パネルにおいて、"Ammo 1" などの汎用的なラベルの代わりに、マスターデータに定義された武器名（例：「戦車砲S」）を表示する。

**Non-Goals:**
- 外部カタログ（gettextなどのi18nライブラリ）の導入。今回はシンプルにハードコード文字列の置換、あるいはモジュール内定数による管理に留める。
- マップ上のアイコン（ユニット記号や地形記号など）の日本語化。現在のASCIIシンボルを維持する。
- エンジン側のロジック変更。

## Decisions

### 1. 文字列の直接置換と定数化
ソースコード内に直接日本語を記述します。
**理由:** 現在のプロジェクト規模において、i18nライブラリを導入するコストがメリットを上回ると判断。
**代替案:** `i18n` クレートの導入。将来的に多言語化（英語に戻すなど）が必要になった場合に検討。

### 2. 半角・全角の使い分け
メニューや短いラベルには全角日本語（例：「移動」）を使用し、操作キー（例：「[Space]」）などのUIガイドの一部には半角英数字を維持します。
**理由:** ターミナル表示における視認性と慣習を考慮。

### 3. 武器名の保持 (Engine層の拡張)
`UnitStats` コンポーネントに `weapon1_name: Option<String>` および `weapon2_name: Option<String>` フィールドを追加します。
**理由:** CLI（プレゼンテーション層）がレンダリング時にマスターデータ（`MasterDataRegistry`）を検索し直すコストと複雑さを避けるため、各ユニット自身に名前を持たせます。
**実装詳細:** ユニット生成時（`produce_unit_system`など）に `MasterDataRegistry` から武器名を取得してコンポーネントに設定します。

## Risks / Trade-offs

- **[リスク] ターミナルでの表示崩れ** → 日本語（全角文字）は半角2文字分の幅をとるため、ratatuiのレイアウト計算（Constraint）で余裕を持たせるように調整します。
- **[トレードオフ] コードの保守性** → 文字列がコード内に分散するため、将来的な多言語化時は一括置換が必要。
