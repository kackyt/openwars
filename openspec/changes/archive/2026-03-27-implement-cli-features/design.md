## Context

現在のCLIバージョンは `crossterm` と `ratatui` を使用したプレゼンテーション層として機能していますが、ユニットの生産（ハードコードされたリストからの選択）やアクションの一部（Drop先選択など）の実装が不完全またはハードコードされています。
本格的にゲームを遊ぶためには、エンジン(`engine` クレート)の状態やマスターデータを正確にUIに反映・操作できる設計が必要です。

## Goals / Non-Goals

**Goals:**
- 各プロパティ（工場・空港・港）のタイプに合わせた生産メニューの動的生成
- ユニットの詳細機能（残HP、弾薬、搭載ユニット等）のインスペクト表示
- 戦闘や各アクションの結果に関するユーザーフィードバックの明確化
- 全てのエンジン機能（Load, Drop, Supply, Join等）を網羅するUIの実装
- **【追加】** 生産と移動・攻撃フェーズの統合（UI体験の向上）
- **【追加】** 正確な射程と陣営のチェック、HP計算とデスポーンへの対応に伴うエンジンとUIの改修

**Non-Goals:**
- `engine` クレートのコアドメインロジックの変更・追加
- GUIクライアント（Tauriなど）向けの機能実装
- マウス操作への完全対応（キーボード操作を前提とする）

## Decisions

**1. 生産メニューの構築方法**
- *Decision*: `ui_state` 内で表示する `ProductionMenu` オプションをプロパティの地形 (`Terrain`) と `loadable_unit_types` などのマスターデータに基づいて構築する。また、対象拠点が自国の首都から4マス以上離れている場合は、「生産不可」のエラーメニューまたは警告を表示する。
- *Rationale*: マスターデータを参照して動的に生成することでメンテナンス性が向上し、ゲームルールをUIで明確に伝えられます。

**2. アクション結果のフィードバック設計**
- *Decision*: 現状のログライン（`log_messages`）表示に加え、重要なイベント（戦闘結果など）はポップアップまたは専用の Notification エリアを設けて一定時間表示する（またはキーボード応答待ちを挟む）。
- *Rationale*: 単なるログ表示だけだとターンの進行とともにながれてしまい、初心者が何が起きたか把握しにくいため。

**3. 複雑なアクション（Dropなど）のUI状態遷移**
- *Decision*: `InGameState` に `CargoSelection` と `DropTargetSelection` のような中間の状態管理を追加する。
- *Rationale*: 搭載されているユニットが複数ある場合、どれを降ろすか選ぶステップが必要なため、既存の状態列挙だけでは対応しきれません。

**4. 移動範囲の視覚化**
- *Decision*: ユニット選択状態 (`InGameState::UnitSelected`) になったタイミングで、`engine` 側の地形コストを参照してダイクストラ法等で到達可能な座標一覧を計算し表示用ステートに保持する。描画時 (`draw_in_game`) にその背景色を変更する。
- *Rationale*: CLI側の描画ループ毎に経路探索を行うと負荷が高いため、選択時のみ計算しキャッシュする設計とします。

**5. 【追加】フェーズの統合と戦闘UIの厳密化**
- *Decision*: ゲームルール上、生産と移動・攻撃フェーズは同列に扱えるため、`Phase::Main` としてフェーズを統合しました。またターゲット選択時に射程チェックを追加し、不正なコマンドの送信を防ぐ。**間接攻撃ユニット（射程の最小値が2以上）については、移動したターンには攻撃できない制約をUIレイヤーでも適用する。** ダメージは `UnitAttackedEvent` に載せてUIパブリッシュし、デスポーン処理もエンジンに委譲する。
- *Rationale*: UIからの不正な入力を防ぎ、正確な戦闘評価ログ（Attacker HP: 10 -> 10）を表示させるため。特に間接攻撃ユニットの挙動を明確にし、UXの混乱を防ぎます。

## Risks / Trade-offs

- [Risk] UIステートの複雑化による不具合 
  - *Mitigation*: `InGameState` の各バリアントを小さく保ち、状態遷移を明確にテスト・ドキュメント化する。
- [Risk] 大量のユニット情報を画面内に収めきれない
  - *Mitigation*: ユニットインスペクト画面は専用のポップアップウィンドウ（Overlay）として実装し、必要な時だけ描画する。
