## ADDED Requirements
### Requirement: ユニットモデル (Unit Models)
システムは、HP（内部的には表示値の10倍、最大100として扱う）、移動力、燃料、弾薬数、移動タイプ（歩行、車両、キャタピラ、タイヤ、低空、高空、艦船など）、および各種武装を持つユニットを定義しなければならない (SHALL)。

#### Scenario: 指定されたユニットの作成
- **WHEN** 「歩兵」「戦闘工兵」「戦車Z」「戦車A」「戦車B」「砲台」「自走砲A」「自走砲B」「対空ミサイル」「対空戦車」「ロケットランチャー」「装甲車」「補給輸送車」「戦闘機A」「戦闘機B」「爆撃機」「レーダー輸送機」「戦闘ヘリ」「輸送ヘリ」「スーパーミサイル」「戦艦」「空母」「輸送船」「潜水艦」のいずれかが作成されたとき
- **THEN** 各ユニット固有のステータス（コスト、移動力、移動タイプ、燃料、武装1/2、射程、補給仕様など）が正しく初期化される

#### Scenario: マスターデータによるダメージ相性の取得
- **WHEN** 攻撃ユニットと防御ユニットの組み合わせに対してダメージを計算するとき
- **THEN** CSV等の外部マスターデータから読み込んだ相性表を参照し、基本ダメージ値を返す

#### Scenario: ダメージの処理とHP制約
- **WHEN** いかなるユニットであってもダメージ計算が行われたとき
- **THEN** プログラム上は表示上のHP(10)を10倍した値として扱い、最大HPは100となる。0を下回った場合は破壊（消滅）として扱われる
