# タスク (Tasks): define-game-rules

1. **[x] Test & Implement: 予算計算ロジック**
   - ターン開始時に、プレイヤーが保有する都市および空港の数に基づいて予算を増加させるロジックを実装する。
   - テスト: 都市/空港を複数所有している状態でターンを開始した際に、適切な額の資金が加算されることを検証する。

2. **[x] Test & Implement: 勝利条件判定ロジック**
   - ターン終了時、またはアクション完了時に、以下の条件を判定してゲームを終了させるロジックを実装する。
     - 条件A: 敵の首都を占領した。
     - 条件B: 敵軍のユニットが0になった（ただし、ゲーム開始直後の1ターン目を除く）。
   - テスト: 各条件を満たしたときに正しく勝敗が決着し、ゲーム状態が「終了」に遷移することを検証する。

3. **[x] Test & Implement: ユニット生産ロジック**
   - プレイヤーが指定のユニットを生産するためのアクションハンドラーを実装する。
   - 生産条件のバリデーションを実装する。
     - 条件1: 生産元が自国の「都市」または「空港」であること。
     - 条件2: その都市または空港が、「自国の首都から3マス以内」に存在すること。
     - 条件3: 生産にかかるコスト以上の資金をプレイヤーが保有していること。
   - 成功時、資金を消費し、指定した位置にユニットを配置する。
   - テスト: 条件を満たす場合と満たさない場合（資金不足、距離が遠い、自国領土でないなど）で正しく生産が許可・拒否されることを検証する。
