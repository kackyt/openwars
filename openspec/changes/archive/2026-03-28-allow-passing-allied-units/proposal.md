# Proposal: allow-passing-allied-units

## Motivation

現状の `MovementSystem` では、移動経路に自軍のユニットが存在する場合（輸送ユニットへの積載時を除き）、そのマスを通過することができません。
これにより、複数のユニットが狭い通路に密集している場合や、後方のユニットが前方のユニットを追い越して移動したい場合に、操作性が著しく低下します。
自軍のユニットであれば互いに道を譲る（通過する）ことができるようにすることで、戦略的な柔軟性とプレイアビリティを向上させることが目的です。

## Change Summary

- `calculate_reachable_tiles` および `find_path_a_star` の経路探索ロジックを修正し、自軍ユニットが占有しているマスを「通過可能」なマスとして扱うようにします。
- ただし、移動の「終点」に自軍ユニット（積載可能な輸送ユニットを除く）がいる場合は、引き続き移動不可とします（重なりは許可しない）。
- 敵軍ユニットについては、引き続き通過不可のままです。

## Impact

- `engine/src/systems/movement.rs` の移動ロジックに影響します。
- `calculate_reachable_tiles`: A*やダイクストラの探索中に、自軍ユニットの位置での中断を解除します。
- `find_path_a_star`: 経路算出において、自軍ユニットが含まれる経路を有効なものとして扱います。
