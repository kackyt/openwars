# 提案: マスターデータ読み込みの実装

## 目的
エンジンは、ユニットや地形といったゲーム内エンティティが持つ属性（最大HP、移動コスト、ダメージ値など）および初期マップデータを外部ファイルから読み込む必要があります。本提案では、`engine/src/resources/master_data/` に存在する既存のCSVファイルを解析し、Bevy ECSの `Resource` として提供するためのマスターデータ読み込みメカニズムを実装します。

## スコープ
* `landscape.csv`、`unit.csv`、`weapon.csv`、`movement.csv`、`load.csv` および `map/` ディレクトリ配下のCSVファイルのパース
* これらのデータを表現するための型安全な構造体/列挙型の定義
* マップデータに対する、所属プレイヤーIDと地形IDを分離させるデコード処理（例: `101` -> プレイヤー1の首都）
* パースされたデータセットを保持するECS `Resource`（例: `MasterDataRegistry`）の作成
* 既存の列挙型にハードコードされたルールの代わりに、ゲームロジックが使用するための参照メソッドの提供

## スコープ外
* 新しいマスターデータコンテンツの作成（新しいユニットや地形の追加など）
* 実行時のマスターデータの動的リロード（ホットリロード）
