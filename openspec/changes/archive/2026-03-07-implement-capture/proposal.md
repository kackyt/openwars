# Proposal: property-capture

## Objective
歩兵ユニットによる拠点（都市、工場、空港、港、首都）の占領（Capture/Occupation）システムを実装します。これにより、ゲームの勝利条件（首都占領）の達成と、毎ターンの収入源となる拠点の獲得が可能になります。

## Current State
- `game-state/spec.md` および `production/spec.md` で言及されている通り、自国の都市や空港に応じた収入や生産、敵首都の占領による勝利条件が定義されています。
- 現在の `GameState` において、`properties` は `HashMap<(usize, usize), u32>` であり、座標に基づく所有者のIDのみを保持しています。占領進行度（耐久値）の概念がありません。
- ユニットが拠点上で「占領」アクションを実行する機能がありません。

## Proposed Capabilities
1. **Property Capture State (拠点の占領状態管理)**
   - 拠点は種類に応じて異なる最大占領耐久値を持ちます。
     - **首都 (Capital)**: 400
     - **空港 (Airport), 港 (Port)**: 300
     - **都市 (City), 工場 (Factory)**: 200
   - 拠点は「自国」「敵国」だけでなく「中立 (Neutral)」の所有状態を持ちます。
   - `GameState` の `properties` の持ち方を変更し、現在の所有者と同時に「現在の耐久値（capture_points）」等を管理できるようにします。
   
2. **Capture Action (占領アクション)**
   - 歩兵および戦闘工兵のみが占領アクションを実行できます。
   - 自国以外の陣営（敵国または中立）の拠点にいる場合に実行可能です。
   - アクションを実行すると、ユニットの現在のHP（1〜10）×10 の分だけ拠点の耐久値を減らし、行動完了状態になります。
   - 耐久値が0以下になった場合、その拠点の所有者が占領プレイヤーに変更され、耐久値がそれぞれの最大値にリセットされます。

3. **Repair Action (修復アクション)**
   - 歩兵および戦闘工兵が自国の拠点の上にいる場合、「修復」アクションを実行できます。
   - アクションを実行すると、ユニットの現在のHP（1〜10）×10 の分だけ拠点の耐久値を回復し（最大値を超えない）、行動完了状態になります。

4. **Persistent Capture Damage (拠点耐久値の保持)**
   - 拠点の耐久値がいったん減少した場合、ユニットが拠点から移動したり、撃破されたり、空き状態になったりしても、耐久値は自動的には回復（最大値へのリセット）しません。
   - 減少した耐久値を回復させる唯一の手段は、自国拠点になった後の「修復（Repair）」アクションのみです。

## Why
オープンウォーズ（Game Boy Wars系ゲーム）において、拠点の確保は戦線の維持と収入増加、ならびに勝利条件そのものに直結する必須の中核システムです。

## Out of Scope
- 本提案では、占領アニメーション等のGUI側の実装は含みません。ロジック（ドメイン層）の実装のみを行います。
- 工場や港での生産については、この提案ではなく別タスク（productionの拡張）で行います。
