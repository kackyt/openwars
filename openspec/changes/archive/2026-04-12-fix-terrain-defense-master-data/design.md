# 設計判断（Design Decisions）: 戦略コアシステム

## 1. 地形防御とダメージ計算

### 1.1 `MasterDataRegistry` の拡張
`landscape.csv` に定義された `defense_bonus` を `Terrain` 列挙型から検索できるメソッドを追加しました。
- 検索に失敗した場合は安全のため `0` を返します。

### 1.2 新しいダメージ計算式
従来の加減算方式を廃止し、地形ボーナスを分母に置く除算方式を採用しました。

- **攻撃側ダメージ**: `(base_damage * attacker_hp + 105) / (100 + terrain_defense_bonus) + luck`
- **反撃側ダメージ**: `(base_damage * defender_hp + 100) / (100 + terrain_defense_bonus) + luck`

※攻撃側を反撃より若干有利にするため、端数処理の定数に差（105 vs 100）を設けています。
※`luck` は 0 から 10 の範囲の整数値が `GameRng` によって加算されます。

## 2. 輸送システム（Transport）

### 2.1 データモデル
- **`CargoCapacity`**: 輸送ユニットが保持する積載量と、現在積載されているユニットのエンティティIDリスト。
- **`Transporting`**: 積載されているユニットに付与されるコンポーネント。親となる輸送ユニットのIDを保持します。

### 2.2 ステート管理
- 積載されたユニットは `GridPosition` をマップ外(9999, 9999)に移動させ、レンダリングや戦闘から一時的に除外します。
- 降車（Unload）時は、輸送ユニットの隣接マスに再配置され、`ActionCompleted` が両方のユニットに適用されます。

## 3. 占領と勝敗判定（Capture & Victory）

### 3.1 占領アルゴリズム
- 拠点の `capture_points`（最大200）を、ユニットの「表示HP × 10」分だけ毎ターン減算します。
- 0になった瞬間に `owner_id` を書き換え、耐久値を全回復させます。
- 同勢力のユニットが拠点上にいる場合は、毎ターン耐久値を回復（修理）します。

### 3.2 勝利条件（Victory Check）
- **勝利**: 全敵プレイヤーの首都（Capital）を占領するか、敵軍を全滅させた場合に確定します。
- **引き分け**: 全プレイヤーが同時に全滅するか、首都を失った場合に発生します。
- **例外**: ターン1では初期配置中であることを考慮し、全滅判定を行いません。

## 4. 高度な移動システム（Movement）

### 4.1 A*アルゴリズムとZOC
- 経路探索に A* を採用し、マンハッタン距離をヒューリスティックに使用します。
- **ZOC (Zone of Control)**: 敵ユニットの隣接マスに進入した場合、経路探索をその地点で強制終了させます。

### 4.2 燃料と墜落
- 移動1タイルごとに燃料を1消費します。
- 航空ユニットは各ターンの開始時（またはフェーズ終了時）に待機燃料を消費します。
- 空中で燃料が0になった場合、ただちにそのユニットを破壊（墜落）させます。空港（Airport）マスにいる場合は墜落を免れ、補給されます。

### 4.3 移動Undo（取り消し）
- `PendingMove` リソースに移動前の「位置」「燃料」を一時保存します。
- アクションが発行（攻撃・補給・待機等）された時点でこのリソースを削除し、移動を確定させます。
- アクション実行前に Undo コマンドが送られた場合、保存されたデータを元に状態を復元します。
