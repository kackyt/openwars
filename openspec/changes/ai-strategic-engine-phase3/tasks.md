## 1. Mission Recovery

- [ ] 1.1 `engine/src/ai/planner.rs` に `validate_missions` メソッドを追加し、World上に存在しない（破壊された）Entityを参照しているミッションを削除する処理を実装する。
- [ ] 1.2 輸送機が破壊された場合、積載されていた歩兵も死亡するためミッション全体を破棄する。歩兵のみが破壊された場合は、ミッションのフェーズを `Pickup` に戻して別の歩兵を待つか、ミッションを破棄して輸送機を解放するロジックを実装する。

## 2. Production Integration

- [ ] 2.1 目標の島に対して必要なユニット数と、現在割り当てられているユニット数を比較し、不足分（Deficit）を計算するロジックを実装する。
- [ ] 2.2 不足分が存在する場合、前線（目標の島に最も近い自軍の島）の工場や空港における `Infantry` や `TransportHelicopter` の生産スコア（Demand）に加点を行うように `production.rs` または `demand.rs` を修正する。
- [ ] 2.3 `map_3` において、輸送ヘリが撃墜された後に、AIが自発的に新しい輸送ヘリを生産し、再度歩兵を乗せて海を渡るリカバリ行動を検証する。
