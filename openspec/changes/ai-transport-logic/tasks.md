## 1. アクション生成（Load / Drop）の拡張

- [ ] 1.1 `engine/src/ai/tactical.rs`（または `action.rs`）において、歩兵ユニットの可能なアクションに `LoadUnitCommand` を含める処理を追加する。
- [ ] 1.2 同じく、搭載済み輸送ユニットの可能なアクションに `DropUnitCommand` を含める処理を追加する。
- [ ] 1.3 `get_valid_actions` の出力で、移動可能な範囲の空き輸送ユニットへの移動＋搭載、および現在地からの降車先が正しく列挙されることを確認する。

## 2. 搭載（Load）アクションの評価実装

- [ ] 2.1 歩兵の `LoadUnitCommand` に対する評価ロジックを追加する。
- [ ] 2.2 目的地が遠く、かつ空き輸送ユニットが近くにある場合に、徒歩移動よりも `Load` アクションのスコアが高くなるように調整する。
- [ ] 2.3 すでに目的地に近い場合は、`Load` のスコアを下げて通常移動や占領を優先させる。

## 3. 降車（Drop）アクションの評価実装

- [ ] 3.1 輸送ユニットの `DropUnitCommand` に対する評価ロジックを追加する。
- [ ] 3.2 目標拠点付近（または前線）での降車に対して高いスコアボーナスを与える。
- [ ] 3.3 降車先が敵の攻撃範囲内で危険な場合は、スコアにペナルティを与える（安全なマスへの降車を優先）。

## 4. 輸送ユニット自体の移動ロジック調整

- [ ] 4.1 空の輸送ユニットが、搭載すべき味方歩兵に向かって移動する（または待機する）ように移動（Move）のスコアを調整する。
- [ ] 4.2 搭載済みの輸送ユニットが、搭乗ユニットの目的に向かって最速で移動するようにスコアを調整する。

## 5. テストと検証
