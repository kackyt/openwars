## 1. 静的盤面評価関数の実装

- [ ] 1.1 `engine`クレートに、指定した勢力の全ユニットの戦力（HPと兵種の基本価値の積）を合算するロジックを追加する
- [ ] 1.2 地形の防御補正や拠点への到達など、位置的な優位性を評価スコアに加算するロジックを追加する
- [ ] 1.3 （必要であれば）予想されるダメージ交換からヘイト評価を計算するロジックを追加する
- [ ] 1.4 これらを総合して特定の勢力にとっての盤面スコアを算出する `evaluate_board` などの主要関数を実装する

## 2. 合法手の枝刈りフィルターの実装

- [ ] 2.1 ユニットが対象ターンで取りうるすべての移動・攻撃パターン（Action）を列挙する機能を実装する
- [ ] 2.2 列挙された行動について、「自分が受ける反撃被害の価値」が「敵に与える被害の価値」を上回る無謀な攻撃行動を候補から除外するフィルター関数を実装する

## 3. 貪欲法による行動決定エンジンの実装

- [ ] 3.1 AIが操作する未行動ユニットを一覧化し、行動順序（優先度）を決定する
- [ ] 3.2 先に決定した順序でユニットを回し、各ユニットの取りうる合法手それぞれについて「仮に行動した場合の盤面」を生成し、`evaluate_board` を実行する
- [ ] 3.3 各ユニットごとに最も盤面スコアが高かった行動（Move, Attack, Waitなどコマンドの形式）を採用し、エンジンシステムへ登録する処理を実装する

## 4. CLI連携とAI自動進行ループの実装

- [ ] 4.1 CLIアプリ側で、現在のターンが「人間」か「AI」かを区別・判定できるようにする
- [ ] 4.2 AIターンの場合、プレイヤーへのキー入力をブロックするかスキップし、自動的にエンジン側のAI行動決定システムを呼び出すフローを構築する
- [ ] 4.3 コマンド実行ごとの描画更新と、すべてのAIユニットが行動を終えたあとの「ターン終了処理」への自動遷移を実装する
- [ ] 4.4 CLIの簡易メニューによって、各陣営（P1, P2）の操作者を「人間(Human)」または「AI」から任意に選択・設定できる対戦モード構築機能を実装する

## 5. 簡易生産AI（デバッグ・フォールバック用）の実装

- [ ] 5.1 人間vs人間などの対戦に影響を与えないよう、エンジンのコアロジック（`systems`）からは切り離し、`engine/src/ai` 内に独立した生産ロジックを実装する
- [ ] 5.2 「工場が空いていて、資金がX以上なら『歩兵』等のユニットを生産する `ProduceUnitCommand` を発行する」というシンプルなAI専用関数を作成する
- [ ] 5.3 ターン開始時（または終了時）に、現在の操作陣営がAIである場合のみ、この生産ロジックが実行されるようにスケジューリングする

## 6. AI検証用 MCP サーバーの構築

- [ ] 6.1 `mcp-server` クレート（またはバイナリ）を新規作成し、標準入出力でLLMと通信する基盤をセットアップする
- [ ] 6.2 `get_board_state` ツール: 盤面状態、ユニット配置、HP等のJSONダンプ機能を実装する
- [ ] 6.3 `evaluate_board` ツール: 現在の盤面に対する戦力スコアや陣地スコアの内訳をJSONで返す機能を実装する
- [ ] 6.4 `get_valid_actions` ツール: 指定したユニットの全行動候補と、AIによる評価スコア（heuristic score）一覧をJSONで返す機能を実装する
- [ ] 6.5 `simulate_ai_turn` ツール: CLIの入力をブロックせず、バックグラウンドでAIに1ターン思考させて結果ログだけを返す機能を実装する
- [ ] 6.6 `execute_action` ツール: 特定のコマンド（ユニットの移動・攻撃、生産など）をMCP経由で直接エンジンに送信し、実際に盤面状態を更新する機能を実装する
