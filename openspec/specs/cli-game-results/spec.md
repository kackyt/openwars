# cli-game-results Specification

## Purpose
ゲームが決着した際に、勝敗結果をユーザーに明示し、円滑に次のプレイ（マップ選択）へ移行できる状態を提供するための仕様を定義します。

## Requirements

### Requirement: ゲーム終了メッセージの表示
勝利条件が満たされた際、中央に結果（例: 勝利、敗北）を表示し、ユーザーが内容を確認できるようにしなければならない (MUST)。

#### Scenario: プレイヤー勝利時の表示
- **GIVEN** アクティブプレイヤーが勝利条件を満たした
- **WHEN** 画面が描画されるとき
- **THEN** 中央のパネルに `PLAYER 1 VICTORY!` （または日本語）と表示されなければならない (SHALL)

### Requirement: タイトル画面（マップ選択）への復帰
ゲーム終了メッセージ表示中にユーザーが特定のキー（例: Enter, Space, Esc）を押下した場合、現在のゲームセッションを破棄し、マップ選択画面（タイトル）に戻らなければならない (MUST)。

#### Scenario: 決定キーでタイトルへ戻る
- **GIVEN** ゲーム終了メッセージが表示されている
- **WHEN** ユーザーが Enter キーを押下したとき
- **THEN** ゲームの状態がリセットされ、再びマップ選択リストが表示されなければならない (SHALL)

#### Scenario: Escキーでタイトルへ戻る
- **GIVEN** ゲーム終了メッセージが表示されている
- **WHEN** ユーザーが Esc キーを押下したとき
- **THEN** ゲームの状態がリセットされ、再びマップ選択リストが表示されなければならない (SHALL)

### Requirement: ゲーム終了モーダルの優先表示
ゲーム終了状態 (`GameOverPopup`) は、他のあらゆる一時的なポップアップ（フェーズ開始通知など）よりも優先して表示されなければならず、他のイベントによって上書きされてはならない (MUST)。

#### Scenario: フェーズ開始通知による上書きのガード
- **GIVEN** ゲーム終了モーダルが表示されている
- **WHEN** フェーズが切り替わり、本来であればフェーズ開始通知ポップアップが表示されるタイミングになったとき
- **THEN** ゲーム終了モーダルが維持され、フェーズ開始通知ポップアップによって上書きされてはならない (SHALL)

### Requirement: モーダルの視覚的な強調
ゲーム終了時のモーダルは、画面中央に配置され、背景のマップを隠すようにオーバーレイ表示されなければならない (MUST)。また、勝敗に応じて枠線の色を変更しなければならない (SHALL)。

#### Scenario: 勝利時のスタイル
- **WHEN** プレイヤー1が勝利したとき
- **THEN** モーダルの枠線またはタイトルが黄色（Yellow）またはシアン（Cyan）で表示されなければならない (SHALL)
