# action-cancellation Specification

## Purpose
本仕様書は、アクションの取り消し（Undo）、キャンセル時のUI状態遷移、武器マスターデータに基づいた射程判定、および移動後の間接攻撃制限に関するゲームルールの詳細を定義します。

## Requirements

### Requirement: 移動取り消し機能
MUST: システムは、ユニットが移動した後にキャンセルが選択された場合、その移動を取り消して（Undo）移動前の座標および燃料状態に戻さなければならない。この際、ユニットの「移動済み（HasMoved）」フラグは解除されなければならない（MUST）。

#### Scenario: アクションメニューでのキャンセル
- **WHEN** ユニットを移動させ、アクションメニュー（Wait, Attack等）が表示されている状態で「Cancel」を選択、または 'x' キーを押す。
- **THEN** ユニットが移動前の座標に戻り、燃料が消費前の値に復元され、`HasMoved` フラグが `false` にリセットされる。

#### Scenario: ターゲット選択画面でのキャンセル
- **WHEN** 「Attack」等のアクションを選択し、ターゲット選択画面（TargetSelection）で 'x' キーを押す。
- **THEN** ターゲット選択が中止され、アクションメニュー（Wait, Attack等）が表示されている状態に戻る。

#### Scenario: ターゲット選択失敗時（エラー含む）
- **WHEN** 「Attack」等のアクションを選択し、ターゲット選択確定時にエラー（射程外、対象なし等）が発生する。
- **THEN** エラーメッセージをログに表示し、`Normal` 状態に戻るのではなく、直前のアクションメニュー状態に復帰する。

### Requirement: 攻撃制限と射程判定
システムは、攻撃実行時に武器の射程およびユニットの移動状態に基づいた制約を課さなければならない (MUST)。
#### Scenario: 武器マスターデータに基づいた射程判定
- **MUST**：システムは、攻撃可能ターゲットを検索する際に、マスターデータの武器性能（最小・最大射程）を参照し、ターゲットとの距離が射程範囲内にある場合のみを「攻撃可能」と判定しなければならない。

#### Scenario: 移動後の間接攻撃制限
- **SHALL**：システムは、最小射程が2以上の武器（間接攻撃武器）において、ユニットがそのターンに移動（座標の変更を伴う移動）を行った場合、攻撃不可と判定しなければならない。

### Requirement: 非移動状態でのキャンセル
SHALL: ユニットを移動させずに「待機」等のアクションを選択する前にキャンセルを行った場合、単にアクションメニューを閉じ、マップ選択状態（Normal）に戻らなければならない。

#### Scenario: 待機状態でのキャンセル
- **WHEN** ユニットを移動させず、現在の座標でアクションメニューを開き、「Cancel」を選択、または 'x' キーを押す。
- **THEN** アクションメニューが閉じ、マップ画面（Normal状態）に戻る。この際、ユニットは引き続き選択可能。

#### Scenario: UIヘルプの表示
- **WHEN** インゲーム画面右側の Info パネルが表示される。
- **THEN** 操作ヘルプの中に `[x] Back/Cancel` という記述が含まれている。
